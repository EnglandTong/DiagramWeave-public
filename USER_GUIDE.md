# DiagramWeave 使用指南

本文面向使用者，说明如何启动 DiagramWeave、绘制流程图、保存和导出文件。

## 1. 启动

### Windows 双击启动

双击项目根目录中的 `start-diagramweave.bat`，或双击中文入口 `启动DiagramWeave.bat`。

首次运行前请先安装官方 Node.js LTS：<https://nodejs.org/>。官方安装包会同时安装 npm。

首次启动会自动完成：

1. 检查 Node.js。
2. 安装项目依赖。
3. 复制本地运行所需的 `vendor/` 文件。
4. 同步中文字体。
5. 启动本地服务并打开浏览器。

### 命令行启动

```bash
pnpm install
pnpm serve
```

然后在浏览器打开：

<http://127.0.0.1:4173/flowchart-editor.html>

不要直接用 `file://` 打开 HTML 文件，否则连线、导入导出或资源加载可能异常。

如果 Windows 提示“智能应用控制已阻止可能不安全的文件”，请参考 [`WINDOWS_START.md`](WINDOWS_START.md) 解除下载文件阻止，或改用命令行启动。

## 2. 基础界面

| 区域 | 用途 |
|---|---|
| 顶部工具栏 | 新项目、选择工具、连线工具、移动画面、撤销重做、自动布局、导入导出、设置 |
| 顶部中间 | 显示当前 Project Name，右侧是演示按钮 |
| 左侧图形栏 | 拖拽流程、判断、文档、数据、存储等图形到画布 |
| 中间画布 | 绘制和编辑流程图 |
| 右侧属性栏 | 编辑当前页面、选中节点或选中连线的属性 |
| 底部提示栏 | 显示常用快捷方式和当前操作提示 |

## 3. 绘制节点

1. 从左侧图形栏拖一个图形到画布。
2. 单击节点选中它。
3. 在右侧属性栏编辑简介、角色、详细说明、耗时、图层、颜色等信息。
4. 拖动节点可调整位置。
5. 选中节点后按 `Delete` 可删除。

常用图形包括：

| 图形 | 常见用途 |
|---|---|
| 流程 | 普通步骤 |
| 判断 | 是/否或多分支判断 |
| 开始/结束 | 流程起点和终点 |
| 文档 | 文档、表单、输出物 |
| 数据 / 存储 | 数据输入、数据库或存储节点 |
| 子流程 | 可复用的子流程步骤 |
| 跨页 | 指向其他页面的跳转节点 |

## 4. 连线

### 创建连线

1. 点击顶部工具栏的连线工具，或按 `L`。
2. 从一个节点边缘的连接点拖到另一个节点。
3. 松开鼠标后创建连线。
4. 选中连线后，可在右侧属性栏编辑条件文字和文字位置。

连接点支持从节点的上、下、左、右方向拉线。使用连线工具时，尽量从想要的方向连接点开始拖拽。

### 编辑连线

| 操作 | 说明 |
|---|---|
| 双击连线文字 | 快速编辑连线条件 |
| 单击连线 | 在右侧属性栏查看和编辑 |
| 拖拽连线端点 | 调整连接到的节点或连接点 |
| 选中后按 `Delete` | 删除连线 |

### 连线路由

顶部工具栏的连线路由下拉可以切换连线方式：

| 模式 | 说明 |
|---|---|
| 平滑曲线 | 适合简单展示 |
| 正交折线 | 使用横平竖直的折线 |
| 加强避障 | 尽量绕开节点 |
| 直线 | 两点直接连接 |
| Visio 连线 | 更接近 Visio 的正交避障连线 |

当多条线交叉时，系统会在交叉处显示桥形标记，表示两条线只是穿过，并不是连接在一起。

## 5. 移动画面和缩放

| 操作 | 说明 |
|---|---|
| 手指工具 | 点击顶部“移动画面”工具，拖动画布整体移动 |
| `H` | 切换到移动画面工具 |
| 按住空格拖拽 | 临时移动画布 |
| 鼠标中键拖拽 | 移动画布 |
| 鼠标滚轮 | 缩放画布 |
| 顶部 `+` / `-` | 放大或缩小 |
| 重置缩放 | 回到 100% 并复位画布位置 |

手指工具只移动整体画面，不会移动节点。

## 6. 表格编辑

点击顶部“表格编辑”工具，或按 `T` 打开表格编辑面板。

| 表格 | 用途 |
|---|---|
| 节点表 | 批量编辑节点编号、简介、角色、形状、说明、耗时、泳道、图层 |
| 连线表 | 批量编辑起点编号、终点编号和条件 |

使用建议：

1. 画布改完后，点击刷新，把当前图同步到表格。
2. 表格改完后，点击应用，把表格内容应用回画布。
3. 删除表格行前，先点击行让它高亮。

## 7. 保存、加载和导出

| 功能 | 用途 |
|---|---|
| 保存工作文件 | 导出 `.diagramweave.json`，保留节点位置、连线、页面和图层 |
| 加载工作文件 | 导入之前保存的 `.diagramweave.json` |
| 保存完整工作文件 Excel | 导出可直接编辑的 `.diagramweave.xlsx`，包含项目、页面、图层、节点、连线等工作表 |
| 加载完整工作文件 Excel | 从 `.diagramweave.xlsx` 恢复图形；也兼容旧版隐藏 JSON 工作簿 |
| 新项目 | 开始一个空白项目；切换前会询问是否先保存当前项目 |
| 设置 | 修改 Project Name 和自动保存间隔，默认每 30 秒自动保存一次 |
| 导出 PNG | 生成图片，适合放入文档或聊天 |
| 导出 SVG | 生成矢量图，适合继续用 Illustrator / Inkscape 编辑 |
| 导出 PDF | 生成适合打印或归档的文件 |
| Excel 模板与导入 | 下载 Excel 模板、导入 Excel 生成流程图、导出当前流程到 Excel |

如果是全新项目，系统会先提示保存新档案；保存后会按设置的间隔自动更新到同一个 `.diagramweave.json` 或 `.diagramweave.xlsx` 文件。打开旧的 JSON / Excel 工作文件时，在支持文件权限的浏览器中会直接自动保存回原文件。

自动保存需要 Edge / Chrome 等支持文件系统权限的浏览器，并建议通过启动 BAT 打开的本机地址使用。如果浏览器不支持该权限，仍可手动保存，系统会改用下载文件的方式。

建议长期编辑时优先保存 `.diagramweave.json` 或 `.diagramweave.xlsx` 完整工作文件，分享或打印时再导出 PNG/SVG/PDF。完整 Excel 工作文件适合需要在 Excel 里直接修改项目资料的用户；普通 Excel 数据表只适合快速批量编辑当前流程的节点和连线。

## 8. 模板

点击左侧“流程模板”按钮可以打开模板选择窗口。

当前仓库包含 `starter-vertical.json`，用于快速生成一个竖向基础流程。应用模板会替换当前画布内容，使用前请先保存当前工作文件。

如需新增模板：

1. 在 `templates/` 目录添加 JSON 模板文件。
2. 在 `templates/index.json` 中登记文件名。
3. 刷新页面后，在模板窗口中选择使用。

## 9. 页面和图层

DiagramWeave 支持多页面和图层。

| 功能 | 说明 |
|---|---|
| 页面 | 可把复杂流程拆成多个页面 |
| 图层 | 可把节点分组管理，便于隐藏、锁定或分层查看 |
| 跨页节点 | 可在流程中表达跳转到其他页面 |

页面名称会影响保存和导出时的默认文件名。建议给重要页面设置清晰名称。

## 10. 演示模式

点击顶部演示按钮进入演示模式。

| 操作 | 说明 |
|---|---|
| 下一步 | 沿流程连线向前演示 |
| 上一步 | 回到上一个步骤 |
| `Esc` | 退出演示模式 |

演示模式会突出当前节点，并在侧边栏显示节点说明、角色、耗时、分支等信息。

## 11. 语言和更新

| 功能 | 位置 |
|---|---|
| 切换中文 / English | 顶部语言下拉，或设置中的界面语言 |
| 同步内容包 | 设置 → 同步内容包 |
| 检查更新 | 设置 → 检查更新 |
| 打开下载页 | 设置 → 打开下载页 |

内容包只更新图标、字体和连线模式配置。主程序更新仍应通过 GitHub Releases 或 Git 更新。

## 12. 常见问题

### 页面打不开

确认已经安装 Node.js 18+，并通过 `pnpm serve` 或启动 BAT 打开页面。

### 提示“未找到 pnpm / npm”

请安装官方 Node.js LTS：<https://nodejs.org/>。如果已经安装 Node.js 但仍然提示缺少 npm，建议重新安装官方 Node.js LTS，并关闭后重新打开 BAT 窗口。

### Windows 阻止 BAT 启动

如果双击 BAT 时出现“智能应用控制已阻止可能不安全的文件”，通常是因为文件来自 GitHub 下载包，被 Windows 标记为来自 Internet。处理方式见 [`WINDOWS_START.md`](WINDOWS_START.md)。

### 端口 4173 被占用

如果本地服务已经启动，浏览器直接打开：

<http://127.0.0.1:4173/flowchart-editor.html>

### 连线或资源加载异常

不要用 `file://` 直接打开 HTML 文件，请通过本地 HTTP 服务访问。

### PDF 中文显示异常

首次启动会同步中文字体。如果 PDF 仍异常，重新运行启动 BAT 或执行：

```bash
pnpm setup
```

### 导入 Excel 失败

优先使用工具内下载的空白 Excel 模板填写。不要修改表头名称，节点编号需要唯一，连线表中的起点编号和终点编号需要能在节点表中找到。
